# 🛡️ АДМИН-ПАНЕЛЬ FLAVORFUSION

## 📋 Обзор

Админ-панель позволяет управлять пользователями и рецептами сайта FlavorFusion. Доступна только пользователям с ролью `admin`.

## 🔐 Доступ

1. **Создание первого администратора:**
   ```bash
   cd backend
   node database/createAdmin.js <username> <email> <password>
   
   # Пример:
   node database/createAdmin.js admin admin@example.com mypassword123
   ```

2. **Вход в админ-панель:**
   - Откройте `http://localhost:5000/admin.html` (или ваш домен)
   - Войдите под учётной записью администратора
   - Система автоматически проверит права и откроет панель

## 🎨 Интерфейс

### Боковое меню
- **Дашборд** — статистика сайта
- **Пользователи** — управление пользователями
- **Рецепты** — управление рецептами
- **На сайт** — переход на главную страницу
- **Выйти** — выход из аккаунта

### Дашборд
Отображает ключевую статистику:
- Всего пользователей
- Всего рецептов
- Заблокированные пользователи
- Всего лайков
- Всего избранных рецептов

## 👥 Управление пользователями

### Возможности:
- ✅ Просмотр списка всех пользователей
- ✅ Поиск по имени или email
- ✅ Фильтрация заблокированных
- ✅ Блокировка/разблокировка
- ✅ Назначение/снятие роли администратора
- ✅ Удаление пользователя (вместе с рецептами)

### Блокировка пользователя:
1. Нажмите кнопку <kbd>🚫</kbd> рядом с пользователем
2. Укажите причину блокировки (необязательно)
3. Подтвердите действие

**Заблокированный пользователь:**
- Не может войти в систему
- При попытке входа видит сообщение о блокировке
- Может просматривать сайт как гость (без авторизации)
- Все действия, требующие авторизации, недоступны

## 📖 Управление рецептами

### Возможности:
- ✅ Просмотр всех рецептов
- ✅ Поиск по названию
- ✅ Фильтрация по категориям
- ✅ Редактирование любого рецепта
- ✅ Удаление рецепта

### Редактирование рецепта:
1. Нажмите кнопку <kbd>✏️</kbd> рядом с рецептом
2. Измените необходимые поля
3. Сохраните изменения

**Изменения сразу отображаются на сайте!**

## 🔧 Технические детали

### API Endpoints

Все endpoints требуют JWT-токен и прав администратора.

#### Дашборд
```
GET /api/admin/dashboard/stats
```

#### Пользователи
```
GET  /api/admin/users?page=1&limit=20&search=&blocked=
POST /api/admin/users/:id/block
POST /api/admin/users/:id/unblock
POST /api/admin/users/:id/make-admin
POST /api/admin/users/:id/remove-admin
DELETE /api/admin/users/:id
```

#### Рецепты
```
GET  /api/admin/recipes?page=1&limit=20&search=&category=
PUT  /api/admin/recipes/:id
DELETE /api/admin/recipes/:id
```

### Безопасность

- Все маршруты защищены middleware `checkAdmin`
- Проверка роли происходит при каждом запросе
- Нельзя заблокировать/удалить самого себя
- Нельзя снять админские права с самого себя

## 🗄️ База данных

В таблицу `users` добавлены поля:
- `role` — роль пользователя (`user` или `admin`)
- `is_blocked` — флаг блокировки
- `block_reason` — причина блокировки

### Миграция
```bash
cd backend

```

## 🚀 Быстрый старт

1. Примените миграцию БД:
   ```bash
   cd backend && npm run migrate
   ```

2. Создайте администратора:
   ```bash
   node database/createAdmin.js admin admin@example.com password123
   ```

3. Запустите сервер:
   ```bash
   npm start
   ```

4. Откройте админ-панель:
   ```
   http://localhost:5000/admin.html
   ```

5. Войдите под созданным администратором

## ⚠️ Важные замечания

1. **Удаление пользователя** — удаляет все его рецепты, лайки, избранное
2. **Блокировка** — не удаляет данные, только ограничивает доступ
3. **Редактирование рецептов** — изменения видны сразу всем пользователям
4. **Первый администратор** — создаётся через скрипт, последующих могут назначать текущие админы

## 📝 Логи действий

Все действия администратора логируются в таблице `activity_log` для аудита.
