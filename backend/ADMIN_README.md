# 🛡️ Админ-панель FlavorFusion

## Обзор

Админ-панель позволяет управлять пользователями и рецептами сайта FlavorFusion.

### Возможности:
- 📊 **Статистика платформы** — общее количество пользователей, рецептов, лайков, подписок
- 👥 **Управление пользователями** — просмотр всех пользователей с их статистикой
- 📖 **Управление рецептами** — просмотр, редактирование и удаление любых рецептов
- 🏆 **Рейтинги** — топ авторов и топ рецептов по лайкам

---

## 🚀 Установка

### 1. Применение миграции базы данных

Добавьте поле `is_admin` в таблицу `users`:

```bash
cd backend
node database/migrate.js
```

Миграция автоматически:
- Применит основную схему (`schema.sql`) с новым полем `is_admin`
- Применит все SQL-файлы из папки `migrations/`

### 2. Назначение администратора

После миграции назначьте права администратора нужному пользователю:

```sql
-- Подключитесь к базе данных PostgreSQL
UPDATE users SET is_admin = TRUE WHERE email = 'ваш@email.com';
```

Или через psql:
```bash
psql -U postgres -d flavorfusion -c "UPDATE users SET is_admin = TRUE WHERE email = 'ваш@email.com';"
```

### 3. Проверка

Убедитесь, что поле добавлено:
```sql
SELECT id, username, email, is_admin FROM users;
```

## 📁 Использование

### Доступ к админ-панели

1. Откройте в браузере: `http://localhost/admin.html` (или путь, где размещён файл)
2. Войдите под учётной записью администратора
3. Если у пользователя нет флага `is_admin`, доступ будет запрещён

### Вкладки админ-панели

#### 📊 Статистика
- Общее количество пользователей, рецептов, лайков, избранного, подписок
- Топ-5 авторов по количеству рецептов
- Топ-5 рецептов по лайкам

#### 👥 Пользователи
Таблица со всеми пользователями:
- ID, аватар, имя, email
- Количество рецептов, полученных лайков, подписчиков
- Дата регистрации
- Кнопка **"Рецепты"** для просмотра всех рецептов пользователя

#### 📖 Рецепты пользователя
При клике на кнопку "Рецепты" открывается список всех рецептов выбранного пользователя:
- Превью рецепта (изображение, название, описание)
- Время приготовления, сложность, количество просмотров
- Кнопки:
  - ✏️ **Редактировать** — открыть модальное окно с формой редактирования
  - 🗑️ **Удалить** — удалить рецепт (с подтверждением)

## 🔧 API Endpoints

Все endpoints требуют JWT-токен в заголовке `Authorization: Bearer <token>` и права администратора.

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `GET` | `/api/admin/users` | Список всех пользователей со статистикой |
| `GET` | `/api/admin/users/:id/recipes` | Рецепты конкретного пользователя |
| `DELETE` | `/api/admin/recipes/:id` | Удалить любой рецепт |
| `PATCH` | `/api/admin/recipes/:id` | Обновить любой рецепт |
| `GET` | `/api/admin/stats` | Общая статистика платформы |

### Пример запроса

```javascript
const token = localStorage.getItem('token');

// Получить всех пользователей
fetch('http://localhost:5000/api/admin/users', {
    headers: {
        'Authorization': `Bearer ${token}`
    }
})
.then(res => res.json())
.then(data => console.log(data));
```

## 🔐 Безопасность

### Middleware защиты

`backend/middleware/admin.js` выполняет:
1. Проверку JWT-токена (через `authMiddleware`)
2. Проверку поля `is_admin` в базе данных
3. Возврат `403 Forbidden` если прав недостаточно

### Важные замечания

- **Никогда не передавайте права администратора ненадёжным пользователям**
- В production замените `cors({ origin: '*' })` на конкретный домен
- Рекомендуется добавить логирование действий администратора

## ❓ Troubleshooting

### Ошибка "Пользователь не найден" после миграции
Проверьте, что миграция применилась:
```sql
\d users  -- в psql покажет структуру таблицы
```

### Ошибка 403 при доступе к админ-панели
Убедитесь, что у пользователя установлен флаг:
```sql
SELECT username, is_admin FROM users WHERE email = 'ваш@email.com';
```

### Ошибка CORS
Если фронтенд и бекенд на разных портах, проверьте настройки CORS в `server.js`:
```javascript
app.use(cors({
  origin: 'http://localhost:3000', // или ваш домен
  credentials: true,
}));
```
