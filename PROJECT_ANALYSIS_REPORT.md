## 🎯 Общая информация о проекте

**FlavorFusion** — веб-платформа (социальная сеть) для публикации, поиска и обмена кулинарными рецептами.

### Ключевые возможности:
- ✅ Публикация рецептов с фото, ингредиентами и инструкциями
- ✅ Просмотр рецептов других пользователей
- ✅ Добавление рецептов в избранное
- ✅ Профили пользователей с аватарами и описанием
- ✅ Фильтрация по категориям и поиск
- ✅ JWT-аутентификация

## 🗄️ База данных (PostgreSQL)

### Таблицы

#### 1. `users` — Пользователи
```sql
id (PK), username (UNIQUE), email (UNIQUE), password_hash,
avatar_url, bio, created_at, updated_at
```

#### 2. `recipes` — Рецепты
```sql
id (PK), author_id (FK → users), title, description, full_description,
image_url, ingredients (JSONB), instructions, time, difficulty,
category, servings, tags, calories, protein, carbs, fat, notes,
views_count, created_at, updated_at
```

#### 3. `favorites` — Избранное
```sql
id (PK), user_id (FK), recipe_id (FK), created_at
UNIQUE(user_id, recipe_id)
```

#### 4. `likes` — Лайки
```sql
id (PK), user_id (FK), recipe_id (FK), created_at
UNIQUE(user_id, recipe_id)
```

#### 5. `subscriptions` — Подписки
```sql
id (PK), follower_id (FK), following_id (FK), created_at
UNIQUE(follower_id, following_id)
CHECK (follower_id != following_id)
```

#### 6. `activity_log` — Активность
```sql
id (PK), user_id (FK), action, recipe_id (FK), recipe_title, created_at
```

### 🔑 Архитектурная особенность
**Данные автора НЕ копируются в таблицу recipes.**  
Рецепты получают данные автора через `INNER JOIN users ON recipes.author_id = users.id`.

**Преимущества:**
- Изменения профиля мгновенно видны во всех рецептах автора
- Отсутствие дублирования данных
- Единый источник правды

## 🔌 API Endpoints

Базовый URL: `http://localhost:5000/api`

### Авторизация (публичные)
| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/auth/register` | Регистрация → JWT |
| POST | `/auth/login` | Вход → JWT |

### Рецепты
| Метод | Путь | Доступ | Описание |
|-------|------|--------|----------|
| GET | `/recipes` | Публичный | Список (пагинация, фильтр, поиск) |
| GET | `/recipes/:id` | Публичный | Один рецепт |
| POST | `/recipes` | 🔒 JWT | Создать рецепт |
| DELETE | `/recipes/:id` | 🔒 JWT | Удалить свой рецепт |
| POST | `/recipes/:id/favorite` | 🔒 JWT | Toggle избранного |
| POST | `/recipes/:id/like` | 🔒 JWT | Лайк рецепта (toggle) |

### Пользователи
| Метод | Путь | Доступ | Описание |
|-------|------|--------|----------|
| GET | `/users/me` | 🔒 JWT | Мой профиль |
| PATCH | `/users/me` | 🔒 JWT | Обновить профиль |
| GET | `/users/me/favorites` | 🔒 JWT | Мои избранные |
| GET | `/users/:id` | Публичный | Публичный профиль + рецепты |

## 📝 Рекомендации

### Технические долги
1. **profile-edit.js** помечен как устаревший (логика перенесена в profile.html)
2. **primer/** — макеты-референсы, не являются частью сайта
3. Требуется настройка загрузки реальных файлов (сейчас только URL)

### Безопасность
1. CORS установлен на `*` — требуется ограничение в production
2. Нет rate limiting для API endpoints
3. Требуется HTTPS для production

### Производительность
1. Нет кэширования популярных рецептов
2. Изображения загружаются по прямым ссылкам без оптимизации
3.可以考虑 добавить CDN для статических ресурсов
